# Musico Teach - Project Handoff Pack

Last updated: 2026-03-07

This folder is a concise handoff for building the (currently unnamed) music teacher/school platform.

## What this includes

- `00_SESSION_HANDOFF.md`: final decisions made so far and what to do next.
- `01_PRODUCT_SPEC.md`: product scope, features, user roles, UX expectations.
- `02_TECH_SPEC.md`: technical architecture (Django-first), tenancy, domains, jobs.
- `03_SITE_DISCOVERY.md`: findings from live exploration of MyMusicStaff.
- `04_BACKLOG_AND_MILESTONES.md`: implementation sequence.
- `05_OPEN_QUESTIONS.md`: unresolved decisions to answer before build.
- `snapshots/`: local copy of UI screenshots used as references.

## Current naming placeholder

- Parent brand placeholder: `Musico`
- Teacher/school app placeholder: `Musico Teach`

## Current direction

- Build one product now (`Teach`), keep `Play` for later.
- Build mobile-first UX for parent/teacher flows.
- Keep heavy admin workflows desktop-optimized.
- Use multi-tenant architecture from day one.
